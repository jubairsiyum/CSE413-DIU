Topic 1
Networking & APIs in Mobile Apps
How mobile apps communicate with the internet, exchange data, and interact with external services.
🌐

Networking

Enabling a mobile app to connect to the internet or other devices to exchange data, download files, and communicate with servers.

🔌

API Calls

Apps interact with APIs (Application Programming Interfaces) through HTTP requests to access external services.

📦

Data Transfer

JSON (JavaScript Object Notation) is the most common data format used for mobile app data exchange.

Types of APIs

REST

Most common. Uses HTTP methods. Stateless. Returns JSON/XML. Used in most mobile apps.

SOAP

XML-based. More rigid structure. Used in enterprise/legacy systems.

GraphQL

Query language. Request exactly the data you need. Flexible, modern.

How an API Works — Step by Step

Step 1

Client Request

Mobile app / browser sends HTTP request to API

→
Step 2

API Processing

Validates API key/token, forwards to server

→
Step 3

Server Processing

Retrieves/updates data from database, prepares response

→
Step 4

Response

JSON/XML returned. Success → data. Fail → error (404, 500)

⚡ Exam Tip
  • APIs are like tollbooths — they control access to external data/services
  • APIs can be public (open) or private (restricted)
  • Always require authentication (API key / token)
Topic 2
RESTful API Deep Dive
The most important API style for mobile development — know every detail.

Definition

REST (Representational State Transfer) is an architectural style for APIs using HTTP requests to access and manipulate data.

Key Characteristics

  • Stateless — each request is independent, no memory of previous requests
  • Uses HTTP Methods — GET, POST, PUT, DELETE
  • Returns JSON/XML — structured data format
  • Resource-based URLs — /users, /products, /orders

HTTP METHODS TABLE

MethodPurposeExample
GETRetrieve dataFetch product list
POSTSend new dataCreate new user
PUTUpdate existingEdit user profile
DELETERemove dataDelete a product

Making API Calls in Flutter (GET Request)

// 1. GET Request — Fetching data import 'package:http/http.dart' as http; import 'dart:convert'; Future fetchData() async { var response = await http.get(Uri.parse('https://jsonplaceholder.typicode.com/posts/1')); if (response.statusCode == 200) { var data = jsonDecode(response.body); print(data); } else { print('Failed to load data'); // Handle errors! } }

Sending Data (POST Request)

// 2. POST Request — Sending data to API Future sendData() async { var response = await http.post( Uri.parse('https://example.com/api/data'), headers: {"Content-Type": "application/json"}, body: jsonEncode({"name": "Alice", "email": "alice@example.com"}), ); print(response.body); }
⚡ Exam Tip — Security
  • Always use HTTPS (not HTTP) for secure communication
  • Authenticate requests with OAuth or JWT tokens
  • Encrypt sensitive data before sending
  • Validate user inputs to prevent attacks (SQL injection, XSS)
Topic 3
JSON — The Data Format
The universal language of data exchange between mobile apps and servers.

What is JSON?

JavaScript Object Notation — a lightweight, text-based format to store and transmit structured data. It is:

  • Human-readable
  • Easy to parse by machines
  • Language-independent
// JSON Structure Example { "person": { "name": "John Doe", "age": 30, "city": "New York" }, "interests": ["coding", "music", "hiking"] }

JSON PARSING IN FLUTTER

// JSON → Dart Object (Parsing) final user = jsonDecode(jsonString) as Map<String, dynamic>; print('Hello, ${user["name"]}!'); print('Email: ${user["email"]}');

JSON Data Types

  • String: "hello"
  • Number: 30, 3.14
  • Boolean: true, false
  • Array: ["a","b","c"]
  • Object: {"key": "value"}
  • Null: null
Topic 4
Firebase — Real-Time Communication
Google's cloud platform for real-time data syncing across all connected devices.
🔥

Firebase Realtime Database

A NoSQL cloud database that syncs data in real-time across all connected clients (devices).

  • Data stored as JSON tree
  • Offline support built-in
  • Listeners trigger on data change
⚡

Real-Time Updates

When data changes on the server, all connected devices are automatically updated — no polling needed.

Use case: Chat apps, live scores, collaborative editing, notifications

Topic 6
Cross-Platform Development
Write once, run anywhere — Flutter & React Native compared.
🐦

Flutter

  • Created by Google
  • Language: Dart
  • Widget-based UI framework
  • Renders with Skia engine (close to native speed)
  • High performance, custom UI
  • Targets: Mobile, Web, Desktop
  • Hot reload ✅
⚛️

React Native

  • Created by Facebook (Meta)
  • Language: JavaScript + React (.jsx)
  • Bridges native components
  • Uses JavaScript Bridge (slight overhead)
  • Large community, native feel
  • Better native integrations
  • Hot reload ✅
FactorFlutterReact Native
Performance⚡ Near-native (Skia)🟡 JS Bridge overhead
Dev SpeedFastFast
UI Consistency✅ Custom, consistentNative look per OS
CostBoth more cost-effective than native
Best ForHigh perf, custom UI, web+desktopJS devs, native integrations

✅ Advantages

  • Code reusability (write once, run anywhere)
  • Faster development & reduced costs
  • Single development team
  • Consistent UI/UX across platforms
  • Hot reload speeds up debugging

❌ Challenges

  • Performance may not match native apps
  • Limited access to platform-specific features
  • UI inconsistencies due to different OS behaviors
  • Dependency on third-party libraries
⚡ Exam Tip — When to Choose
  • Choose Flutter → High performance, custom UI, targeting web & desktop too
  • Choose React Native → You prefer JavaScript, need large community, better native integrations
  • Choose Native → Maximum performance, full OS feature access, complex platform-specific UI

// Function to test: bool isPrime(int number) { if (number <= 1) return false; for (int i = 2; i < number; i++) { if (number % i == 0) return false; } return true; } // Unit Test: void main() { test('Prime number test', () { expect(isPrime(2), true); expect(isPrime(4), false); expect(isPrime(11), true); expect(isPrime(1), false); }); }

Debugging Strategies & Tools

Reproduce

Identify exact steps to trigger the bug.

Isolate

Narrow down to specific function/component causing the issue.

Fix & Verify

Apply the fix and run tests to confirm resolution.

Logcat (Android)

Command-line tool for viewing system logs and debugging Android apps.

Xcode Debugger (iOS)

Powerful debugger for iOS apps, integrated into Xcode.

Flutter DevTools

Suite for Flutter apps — performance profiling and widget inspection.

Performance Testing

  • Tests app under different conditions (low memory, high CPU, heavy load)
  • Tools: JMeter, Firebase Performance Monitoring
  • If API takes > 2 seconds → needs optimization
  • If app crashes under load → memory leak or inefficiency

App Deployment & Maintenance
Getting your app to users — Google Play Store and Apple App Store submission process.
🤖

Google Play Store

1. Prepare the App

Test thoroughly. Create signed APK or App Bundle.

2. Create Developer Account

One-time fee: $25

3. Upload the App

App name, description, screenshots, APK/Bundle

4. Set Pricing & Distribution

Free or paid. Select target countries.

5. Submit for Review

Google reviews for policy compliance.

🍎

Apple App Store

1. Prepare the App

Test on real devices. Create archive using Xcode.

2. Create Developer Account

Annual fee: $99/year

3. Upload the App

Use Xcode or App Store Connect

4. Set Pricing & Availability

Choose pricing tiers. Select target regions.

5. Submit for Review

Apple reviews for HIG compliance. Stricter than Google.
